Message-Oriented Middleware (MOM)

Background Knowledge

Messaging

  • Definition: Messaging is a method of communication used among software components or applications.
  • Communication Style: It utilizes a peer-to-peer facility where a messaging client has the ability to send messages to, and receive messages from, any other client.
  • Mechanism: Each client connects to a ‘messaging agent’. This agent facilitates the process of creating, sending, receiving, and reading messages.
  • Features of Distributed Communication:
    • Loose Coupling: Messaging supports distributed communication that isn’t tightly linked. This means that each component in the system can operate somewhat independently from the others.
    • Asynchronous Nature: The sender of a message and its receiver don’t need to operate or be available simultaneously. A component can send a message to a designated location (or destination), and the intended recipient can fetch the message from that location whenever convenient.

  • Advantages of Messaging:

    • Heterogeneous (multiple) Integration

    • Reduce System Bottlenecks

    • Increase Scalability

In simpler terms, messaging is a way for software components or applications to communicate in a flexible, loosely tied manner. It’s built around the idea that the communicating parties don’t need to be active or present at the same instant to exchange information.

Messaging Services

  • Definition: Messaging services are functionalities within software components that are based on a service-oriented architecture. They enable users to manage (create, send, receive, read, delete, and save) messages in various forms such as text, audio, pictures, and video. The communication takes place between different software components.
  • Communication Style: It operates using a peer-to-peer (P2P) communication model.
  • Client-Server Dynamics: Each software component in this setup can act both as a client (sending messages) and a server (receiving messages).
  • Data Transmission: To send a message, the sender doesn’t need intricate knowledge about the receiver, just the destination and the data format.
  • Loose Coupling: The communication between components is loosely coupled, which means that they are not heavily interdependent and can function independently.
  • Type:
    • Cloud Messaging Service (CMS): To communicate with varied, distributed systems, the cloud messaging service often relies on middleware. MOM is a common type of middleware employed in cloud messaging services to ensure seamless communication between different systems.
    • Short Message Service (SMS)
    • Instant Message Service (IMS)
    • Multimedia Message Service (MMS)

In simple terms, messaging services are systems or platforms that allow different software components to exchange various types of messages with each other, using a model where each component can be both a sender and a receiver.

Message-Oriented Middleware (MOM)

==Message-oriented Middleware (MOM) is a software or hardware that provides a communication channel between heterogeneous applications, platforms and systems by passing messages across between the communicating systems.==

Types of MOM

==There are two types of MOM: Message Queuing (Point-to-Point) and Publish/Subscribe.==

Message Queuing (Point-to-Point)

  • Definition: ==In Message Queuing (Point-to-Point) model, messages are sent to specific queues, and a receiver pulls the message from that queue for processing. If multiple receivers are listening to the same queue, only one of them will receive and process a particular message.==

Publish/Subscribe

  • Definition: ==In Publish/Subscribe model, messages are sent to topics, not to specific receivers. Subscribers express interest in one or more topics and only receive messages that are of interest to them.==

  • Types of Model: ==There are two types of Publish/Subscribe model, Topics-Based and Content-Based.==

    • ==Topics-Based:==
      1. ==Uses a hierarchical naming scheme for topics.==
      2. ==Topics are logically arranged based on some form of classification method.==
      3. ==Specific terms are used to describe the topics, such as “Political”, “Mathematics”, “Scientific”, etc.==
      4. ==Each topic typically corresponds to dedicated queues to handle messages pertaining to those topics.==
      5. ==Being tied to specific terms makes this method somewhat restrictive.==
    • ==Content-Based:==
      1. ==Does not rely on specific terms to categorize messages.==
      2. ==Uses a query-like syntax for subscribers to define their interests based on message content.==
      3. ==This approach is more dynamic and flexible, allowing for complex conditions and criteria to filter messages.==

MOM Queuing Systems

There are five types of Queuing Systems in MOM:

  1. ==First-In-First-Out (FIFO)==

    ​ ==The first message sent will be the first to be received, and the last message sent will be the last to be received. It follows the principle that messages are processed in the order they arrive.==

  2. ==FIFO with Priorities==

    ​ Operates similarly to the FIFO system, but the order in which messages are sent/received depends on the priority set for both the publisher and the consumer. Messages with higher priorities can be processed before those with lower priorities, even if they arrive later.

  3. ==Journal Queue==

    ​ ==The system maintains a track and a copy of all messages. This ensures that there is a record of each message, which can be crucial for auditing, recovery, and ensuring data integrity.==

  4. ==Public Queue==

    ​ This type of queue is open for access to all users. Any publisher can send messages to it, and any consumer can retrieve messages from it.

  5. ==Private Queue==

    ​ Access to this type of queue requires authentication and authorization. Only approved publishers can send messages to it, and only authorized consumers can retrieve messages from it. It’s more secure compared to a public queue.

  6. ==Connector/Bridge==

    ​ Serves as a proxy to connect to commercial MOM’s queues. It allows integration and communication between different MOM systems or different instances of the same MOM.
